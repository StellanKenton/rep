| 字段 | 字节数 | 说明 |
| --- | --- | --- |
| Header | 2 | 固定为 `0xFA 0xFC` |
| Version/Type | 1 | 当前固定写 `0x01` |
| CMD | 1 | 命令字 |
| Length | 2 | 大端，表示 payload 长度 |
| Payload | N | 业务数据 |
| CRC16 | 2 | 对 `RevData[3]` 起始的数据做 CRC16 |

```txt
1. 0x01 握手帧

发送方 (Send):​ HU (上位机)

接收方 (Receive):​ MCU (下位机)

数据域 (Data) 内容:

Byte 0:​ 帧头 (Head)，固定值 0xFA。

Byte 1:​ 帧头 (Head)，固定值 0xFC。

Byte 2:​ 协议版本 (Ver)，固定值 0x01。

Byte 3:​ 命令字 (CMD)，固定值 0x01(代表握手命令)。

Byte 4:​ 数据长度高位 (Length_H)，表示整个帧数据部分（从Byte5开始）的长度高字节。

Byte 5:​ 数据长度低位 (Length_L)，表示整个帧数据部分（从Byte5开始）的长度低字节。

Byte 6~21:​ 握手密文。这是设备MAC地址经过AES-128加密后的密文，固定占据16个字节。

备注 (Note):​ 上位机在蓝牙连接成功后，必须在10秒内发送此包进行握手与认证。

2. 0x01 握手回复帧

发送方 (Send):​ MCU (下位机)

接收方 (Receive):​ HU (上位机)

数据域 (Data) 内容:

Byte 0:​ 帧头，固定值 0xFA。

Byte 1:​ 帧头，固定值 0xFC。

Byte 2:​ 协议版本，固定值 0x01。

Byte 3:​ 命令字，固定值 0x01(握手回复)。

Byte 4:​ 数据长度高位 (Length_H)。

Byte 5:​ 数据长度低位 (Length_L)。

Byte 6~17:​ 设备原始MAC地址。以明文形式存放，为12个十六进制字符（大写），例如 “A1B2C3D4E5F6”。

备注 (Note):​ 仅当下位机认证通过时，才会发送此响应帧。

3. 0x03 心跳包帧

发送方 (Send):​ HU (上位机)

接收方 (Receive):​ MCU (下位机)

数据域 (Data) 内容:

Byte 0:​ 帧头，固定值 0xFA。

Byte 1:​ 帧头，固定值 0xFC。

Byte 2:​ 协议版本，固定值 0x01。

Byte 3:​ 命令字，固定值 0x03(代表心跳命令)。

Byte 4:​ 数据长度高位 (Length_H)。

Byte 5:​ 数据长度低位 (Length_L)。对于此命令，数据部分长度为0，故 Length_H 和 Length_L 均为 0x00。

备注 (Note):​ 上位机每秒发送一次，用于保活，确保连接在线。

4. 0x03 心跳包回复帧

发送方 (Send):​ MCU (下位机)

接收方 (Receive):​ HU (上位机)

数据域 (Data) 内容:

Byte 0:​ 帧头，固定值 0xFA。

Byte 1:​ 帧头，固定值 0xFC。

Byte 2:​ 协议版本，固定值 0x01。

Byte 3:​ 命令字，固定值 0x03(心跳回复)。

Byte 4:​ 数据长度高位 (Length_H)。

Byte 5:​ 数据长度低位 (Length_L)。

Byte 6:​ 电池电量。范围 0~5，共6个等级。0~1为低电量（设备显示红色），2~5为正常电量（设备显示绿色）。

Byte 7~8:​ 电池电压。表示电池实际电压，范围 0~500，单位 0.01V。Byte7 为高字节，Byte8 为低字节。

Byte 9:​ 电池充电状态。

0x00: 无电池

0x01: 低电量

0x02: 正在充电

0x03: 正常电量

Byte 10:​ 电极片在位标志。

0x00: 没有电极片插入

0x01: 有电极片插入

Byte 11:​ 电极片型号。

0x01: 电极片1

0x02: 电极片2

0x03: 电极片3

Byte 12:​ 电极片贴合状态。

0x00: 电极片未贴合

0x01: 电极片贴合

备注 (Note):​ 下位机收到上位机的心跳包后回复此帧，表示设备在线。

5. 0x04 断开连接命令

发送方 (Send):​ HU (上位机)

接收方 (Receive):​ MCU (下位机)

数据域 (Data) 内容:

Byte 1:​ 帧头，固定值 0xFC。

Byte 2:​ 协议版本，固定值 0x01。

Byte 3:​ 命令字，固定值 0x04(断开连接)。

Byte 4:​ 数据长度高位 (Length_H)。

Byte 5:​ 数据长度低位 (Length_L)。数据部分长度为0。

备注 (Note):​ 由上位机发出，发送此命令后下位机会主动断开蓝牙连接。

7. 0x05 自检信息上报

发送方 (Send):​ MCU (下位机)

接收方 (Receive):​ HU (上位机)

数据域 (Data) 内容:

Byte 0:​ 帧头，固定值 0xFA。

Byte 1:​ 帧头，固定值 0xFC。

Byte 2:​ 协议版本，固定值 0x01。

Byte 3:​ 命令字，固定值 0x05(自检信息上报)。

Byte 4:​ 数据长度高位 (Length_H)。

Byte 5:​ 数据长度低位 (Length_L)。

Byte 6:​ CPR模块自检结果。每个Bit位代表一个状态：

BIT0:​ 自检通过。1表示通过，0表示未通过。

BIT1:​ 加速度计初始化失败。对应故障ID E01，故障等级 H(高)。

BIT2:​ RTC自检错误。对应故障ID E02，故障等级 M(中)。

BIT3~7:​ 保留 (Reserve)。

Byte 7:​ 电源模块自检结果。每个Bit位代表一个状态：

BIT0:​ 自检通过。1表示通过，0表示未通过。

BIT1:​ 3.3V电源过高。对应故障ID E11，故障等级 H。

BIT2:​ 3.3V电源过低。对应故障ID E12，故障等级 H。

BIT3:​ 5V电源过高。对应故障ID E13，故障等级 H。

BIT4:​ 5V电源过低。对应故障ID E14，故障等级 H。

BIT5:​ DC电压输入过高。对应故障ID E15，故障等级 H。

BIT6~7:​ 保留 (Reserve)。

Byte 8:​ 通信模块自检结果。每个Bit位代表一个状态：

BIT0:​ 自检通过。1表示通过，0表示未通过。

BIT1:​ 无线模块初始化失败。对应故障ID E31，故障等级 M(中)。

BIT2~7:​ 未在文档中定义。

备注 (Note):​ 由上位机发出，下位机收到后会回复“上传设备信息”帧。

9. 0x11 上传设备信息

发送方 (Send):​ MCU (下位机)

接收方 (Receive):​ HU (上位机)

数据域 (Data) 内容:

Byte 0:​ 帧头，固定值 0xFA。

Byte 1:​ 帧头，固定值 0xFC。

Byte 2:​ 协议版本，固定值 0x01。

Byte 3:​ 命令字，固定值 0x11(上传设备信息)。

Byte 6:​ 设备类型。数值范围 0~4。

Byte 20~23:​ 软件版本号。4个字节，例如 1.0.0.0对应的四个字节。

11. 0x31 CPR数据帧

发送方 (Send):​ MCU (下位机)

接收方 (Receive):​ HU (上位机)

数据域 (Data) 内容:

Byte 0:​ 帧头，固定值 0xFA。

Byte 1:​ 帧头，固定值 0xFC。

Byte 2:​ 协议版本，固定值 0x01。

Byte 3:​ 命令字，固定值 0x03(心跳回复)。

Byte 4:​ 数据长度高位 (Length_H)。

Byte 5:​ 数据长度低位 (Length_L)。

byte6~9：时间戳：设备从上电到现在的时间，单位ms。

byte10~11：按压频率：CPR算法算出的按压频率，数值范围0~200，含义为每分钟按压的预估次数，byte10为高字节。

byte12：按压深度：CPR算法算出的按压深度，数值范围0~80，单位0.1cm。

byte13：回弹深度：CPR算法算出的回弹的深度，数值范围0~80，单位0.1cm。

byte14：中断时长：从不再按压后开始计数，数值范围0~60，单位s。

byte15~18：事件ID：CPR按压的事件ID，内容为开机累计次数，每次开机时才会+1。
```
