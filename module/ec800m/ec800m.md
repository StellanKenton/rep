# EC800M 模块说明

这是当前目录的权威入口文档。

## 1. 模块定位

`ec800m` 是面向 Quectel EC800M-CN 的轻量 AT 模块 core。当前只覆盖模块启动、通用 AT 事务、MQTT 和 HTTP 命令组包，不在 core 中绑定具体 USB host、GPIO 或业务 manager。

当前边界如下：

- `flowparser` 负责 AT 文本事务、prompt 和 done/error/timeout。
- `ec800m.c` 负责设备实例、启动状态机、公共 API 和后台收发。
- `ec800m_http.c` 负责 HTTP URL、POST、READ 等命令组包。
- `ec800m_mqtt.c` 负责 MQTT open/connect/subscribe/publish 命令组包。
- transport、tick、pwrkey/reset 控制由项目侧 `User/port` 注入。

## 3. 对外公共接口

稳定公共头文件：`ec800m.h`

稳定 API：

- `ec800mGetDefCfg()` / `ec800mGetCfg()` / `ec800mSetCfg()`
- `ec800mInit()` / `ec800mStart()` / `ec800mStop()` / `ec800mProcess()`
- `ec800mIsReady()` / `ec800mGetInfo()` / `ec800mGetState()`
- `ec800mSubmitTextCommandEx()` / `ec800mSubmitPromptCommandEx()`
- `ec800mSetUrcHandler()` / `ec800mSetUrcMatcher()`
- `ec800mHttpBuild*()` 和 `ec800mMqttBuild*()` helper

## 4. 生命周期与 ready 条件

启动流程按下面阶段推进：

1. reset 拉有效并释放。
2. pwrkey 拉有效并释放。
3. 在 ready deadline 内轮询 `AT`。
4. `ATE0` 关闭 echo。
5. 查询 `AT+CPIN?` 和 `AT+CSQ`。
6. 进入 `READY`，允许上层提交 MQTT/HTTP 事务。

ready 条件：

- transport 初始化成功。
- control hook 至少能控制 pwrkey/reset。
- `AT` 探测成功。
- 基础 SIM 和信号查询事务完成；查询失败会进入 `ERROR`，由上层 stop/start 或重新 init 恢复。

## 5. 函数指针 / port / assembly 契约表

| 名称 | 必需/可选 | 由谁实现 | 在哪里被调用 | 成功语义 | 失败语义 |
| --- | --- | --- | --- | --- | --- |
| `ec800mLoadPlatformDefaultCfg` | 必需 | `User/port` | `ec800mGetDefCfg` | 写入默认 linkId、控制脚和时序 | 空实现会导致默认 linkId 不可用 |
| `ec800mGetPlatformTransportInterface` | 必需 | `User/port` | `ec800mInit`、后台收发 | 返回 init/write/getRxLen/read/getTickMs | 返回 NULL 时 `Init` 返回 `NOT_READY` |
| `ec800mGetPlatformControlInterface` | 必需 | `User/port` | `ec800mInit`、`ec800mStart` | 返回 pwrkey/reset 控制接口 | 缺失时 `Start` 返回 `UNSUPPORTED` |
| `ec800mPlatformIsValidCfg` | 建议 | `User/port` | `Get/Set/Init` | 校验当前工程允许的 linkId/pin | false 时返回 `INVALID_PARAM` |

## 6. 公共函数使用契约表

| 来源模块 | 公共函数 | 允许在哪些文件调用 | 用途 |
| --- | --- | --- | --- |
| `flowparser_stream` | `flowparserStreamInit/Reset/Feed/Submit/Proc` | `ec800m.c` | 文本事务、prompt 事务、URC 派发 |
| transport hook | `init/write/getRxLen/read/getTickMs` | `ec800m.c` | 把 USB CDC/ACM 或其他链路映射为 AT 字节流 |

## 7. 改动落点矩阵

| 需求 | 应改文件 | 不该改的文件 |
| --- | --- | --- |
| 改 EC800M 启动顺序或 URC 判定 | `ec800m.c/.h`、`ec800m_ctrl.h` | 业务 manager |
| 改 HTTP/MQTT AT 命令文本 | `ec800m_http.*`、`ec800m_mqtt.*` | USB BSP |
| 改默认 USB linkId 或控制脚 | 项目侧 `ec800m_port.*` | `ec800m.c` |
| 改 USB Host CDC/ACM 枚举和端点收发 | 项目侧 `bspusb.*` 或 `bspusb_ec800.*` | `rep/module/ec800m` |

## 8. USB 连接约束

EC800M-CN 的 USB 口作为 USB device 暴露，当前 MCU 侧必须提供 USB Host CDC/ACM 或 Quectel 复合设备 AT 口绑定。若底层 `drvusb` 仍是 device-only 或返回 `DRV_STATUS_UNSUPPORTED`，`ec800mInit()` 会失败，不能认为 USB 已经连通。

## 9. 验证清单

- `ec800mInit()` 能成功初始化 transport。
- `ec800mStart()` 后持续调用 `ec800mProcess()`，状态能进入 `READY`。
- `AT`、`ATE0`、`AT+CPIN?`、`AT+CSQ` 事务均返回 `OK`。
- 使用 `ec800mMqttBuildPublishExCommand()` 提交 prompt publish 时能收到 `OK` 或 `+QMTPUBEX` 结束行。
- 使用 `ec800mHttpBuildPostCommand()` 提交 prompt POST 时能收到 `CONNECT` prompt 和后续 `OK`/`+QHTTPPOST`。
