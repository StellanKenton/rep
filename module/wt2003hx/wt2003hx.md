# WT2003HX 模块说明

这是当前目录的权威入口文档。

## 1. 模块目标

`wt2003hx` 封装音频模块的串口协议、命令组帧、接收帧解析和状态缓存。它属于可复用器件模块，只关心“WT2003HX 怎么工作”，不关心当前板子的 UART、GPIO 或 CPR 业务播放时机。

## 2. 功能边界

| 归属 | 内容 |
| --- | --- |
| module | 帧格式 `7E len cmd payload checksum EF`、命令码、校验和、查询/播放/停止/音量/模式/输出设置、回复状态缓存 |
| port | 默认 UART linkId、使能 GPIO、使能电平、上电/下电延时、tick 来源 |
| manager | CPR 开始音、滴声节拍、按压质量提示、低电提示、语言/音量设置持久化策略 |

## 3. 文件分工

| 文件 | 职责 |
| --- | --- |
| `wt2003hx.h` | 公共命令、状态、配置、上下文和 API |
| `wt2003hx_assembly.h` | transport/control hook 契约 |
| `wt2003hx.c` | 协议组帧、`frameparser` 配置 provider、接收处理和缓存更新 |
| `wt2003hx.md` | 当前目录 contract |

## 4. 函数指针 / port / assembly 契约表

| 名称 | 必需 | 由谁实现 | 用途 |
| --- | --- | --- | --- |
| `wt2003hxTransportInitFunc` | 是 | `User/port` | 初始化 UART 接收与发送能力 |
| `wt2003hxTransportWriteFunc` | 是 | `User/port` | 发送完整命令帧 |
| `wt2003hxTransportGetRxLenFunc` | 是 | `User/port` | 查询 UART 已缓存字节数 |
| `wt2003hxTransportReadFunc` | 是 | `User/port` | 读取 UART 字节流并交给 `frameparser` |
| `wt2003hxTransportGetTickMsFunc` | 是 | `User/port` | 供模块记录超时和节流 |
| `wt2003hxControlSetEnableFunc` | 是 | `User/port` | 控制模块使能脚 |
| `wt2003hxControlDelayMsFunc` | 是 | `User/port` | 上电/下电等待 |

## 5. 公共函数使用契约表

| 函数 | 调用方 | 前置条件 | 说明 |
| --- | --- | --- | --- |
| `wt2003hxInit()` | port/manager | 已能访问 UART 与 GPIO hook | 初始化 UART、复位使能脚、初始化 `frameparser` |
| `wt2003hxProcess()` | manager 周期任务 | `wt2003hxInit()` 成功 | 读取 UART 字节流并消费完整回复帧 |
| `wt2003hxPlayName()` | manager | `isReady == true` | 按不带扩展名的文件名播放，长度 1-8 |
| `wt2003hxQuery()` | manager | `isReady == true` | 查询版本、音量、状态、曲目数或连接状态 |
| `wt2003hxGetInfo()` | manager | 任意时刻 | 获取缓存状态快照 |

## 6. 使用示例

```c
if (wt2003hxInit(WT2003HX_DEV0) == WT2003HX_STATUS_OK) {
    (void)wt2003hxSetVolume(WT2003HX_DEV0, 26U);
    (void)wt2003hxPlayName(WT2003HX_DEV0, (const uint8_t *)"Z1", 2U);
}

for (;;) {
    (void)wt2003hxProcess(WT2003HX_DEV0);
}
```
