# PCA9535 模块设计说明

## 1. 模块目标

`pca9535` 是一个寄存器型 IO 扩展器模块。它对上提供初始化、输入读取、输出配置、极性配置和方向配置接口；对下通过 `pca9535_port.c` 适配硬件 IIC 或软件 IIC。

当前工程默认把 `PCA9535_DEV0` 绑定到 `DRVANLOGIIC_PCA`，对应板级引脚为：

- `PB3` -> `PCA9535_SDA`
- `PB4` -> `PCA9535_SCL`
- `PA15` -> `PCA9535_RESET`

## 2. 文件分工

### 2.1 `pca9535.h`

负责定义：

- 逻辑设备编号 `ePca9535MapType`
- 地址、寄存器、状态码宏
- IIC port 接口表和绑定结构
- 配置结构 `stPca9535Cfg`
- 运行时结构 `stPca9535Device`
- 对外公共 API

### 2.2 `pca9535.c`

负责实现：

- 默认配置懒加载
- 参数和绑定合法性检查
- 初始化流程
- 单寄存器和 16bit 端口寄存器读写
- 输入、输出、极性、方向的公共接口

core 只依赖 port 提供的三类 IIC 原子动作：

- `init`
- `writeReg`
- `readReg`

### 2.3 `pca9535_port.h`

负责声明：

- 默认配置 / 默认绑定接口
- 绑定切换接口
- 绑定合法性检查接口
- reset 控制接口
- 毫秒延时接口

### 2.4 `pca9535_port.c`

负责实现：

- 默认配置数组
- 软件 IIC / 硬件 IIC adapter
- reset 引脚初始化与拉低/释放
- RTOS/裸机共用毫秒延时

## 3. 初始化流程

`pca9535Init()` 按下面顺序执行：

1. 获取设备上下文并校验配置。
2. 初始化底层 IIC。
3. 如果启用了 `resetBeforeInit`，拉低 `RESET` 后再释放。
4. 先写输出寄存器，避免切换方向时出现错误电平。
5. 写极性寄存器。
6. 写方向寄存器。
7. 更新缓存并置 `isReady = true`。

## 4. 默认配置

当前默认值放在 `gPca9535PortDefCfg` 中：

- 默认总线：`DRVANLOGIIC_PCA`
- 默认地址：`PCA9535_IIC_ADDRESS_HLL`
- 默认输出值：`0xFFFF`
- 默认极性：`0x0000`
- 默认方向：`0xFFFF`
- 默认上电初始化前执行硬件复位

这里把方向默认成全输入，是为了让 module 保持通用，具体哪些位要切成输出应由上层业务决定。
